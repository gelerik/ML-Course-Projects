### 1. Собственная реализация Логистической регрессии
[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/gelerik/ML-Course-Projects/blob/main/Native_Logistic_Regression_with_L2_Regularization.ipynb)

* **Описание:** Собственная реализация логистической регрессии, написанная на Python с использованием `NumPy`. Модель использует пакетный градиентный спуск Mini-batch SGD, поддерживает L2-регуляризацию. В проекте проведено сравнение собственной реализацией логистической регресии с соответсвующей моделью из `scikit-learn`.
* **Стек:** Python, NumPy, Matplotlib, scikit-learn
* **Итоги проделанной работы:**
* **Модель**: Модель логистической регресии с L2 регулязацией успешно реализована с нуля.
* **Метрики:** Точность собственного алгоритма (Accuracy = 0.68) на тестовой выборке сопоставима с эталонной моделью из scikit-learn (Accuracy = 0.70).
* **Сходимость:** График функции потерь демонстрирует плавное падение, доказывая корректность написанного градиентного спуска.
* **Анализ:** Проведено сравнение итоговых весов двух моделей.

### 2. Задача кредитного предложения (Credit offer)
[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/gelerik/ML-Course-Projects/blob/main/Credit_offer_acceptance_prediction.ipynb)

* **Описание:** Бизнес-задача прогнозирования вероятности того, что клиент примет предложение банка и оформит кредит.
* **Стек:** Python, Pandas, numpy, scikit-learn, RandomForestClassifier, KNNImputer, OneHotEncoder, Matplotlib.
* **Итоги проделанной работы:**
1. Подготовил данные: Настроил импорт и обработал пропуски с помощью KNNImputer, обеспечив чистоту выборки.
2. Провел Feature Engineering: Очистил названия регионов через регулярные выражения и применил Target Encoding для работы с географией.
3. Закодировал категории: Использовал OneHotEncoder, чтобы перевести категориальные признаки в понятный для модели формат без ложных зависимостей.
4. Устранил утечку данных: Нашел и удалил признак overdue_credit_count (т.к. кореляция с целевым признаком была 1), что позволило получить честный ROC-AUC.
5. Обучил и валидировал модель: Настроил RandomForestClassifier с балансировкой весов и подтвердил его эффективность на тестовых данных.

### 3. Задача предсказания оттока клиентов по данным телеком компании
[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/gelerik/ML-Course-Projects/blob/main/Telecom_сustomer_сhurn_prediction.ipynb)

* **Описание:** Обучить модель для предсказания оттока клиентов телекоммуникационной компании.
* **Стек:** Python, scikit-learn, CatBoost, seaborn, plotly, ydata-profiling
* **Итоги проделанной работы:**
 1. Очистил данные: Нашел скрытые пропуски в TotalSpent, преобразовал колонку в числа и заполнил пустоты нулями.
 2. Провел EDA: Выявил дисбаланс классов и обнаружил, что новички — самая уязвимая группа для оттока.
 3. Настроил визуализацию: Подключил автоматические отчеты и интерактивные графики для поиска скрытых зависимостей.
 4. Построил Baseline: Разработал пайплайн с логистической регрессией и оптимизировал гиперпараметры через GridSearchCV.
 5. Обучил CatBoost: Использовал мощный бустинг с нативной поддержкой категорий, что позволило достичь ROC-AUC 0.8523.
